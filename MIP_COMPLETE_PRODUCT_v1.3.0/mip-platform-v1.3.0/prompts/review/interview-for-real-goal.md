# Interview for the Real Goal

Before designing a broad modernization change, conduct a focused requirements interview.

Bias toward small, compartmentalized specifications.

Ask only questions that change:

- scope
- observable behavior
- interfaces
- data semantics
- target architecture
- verification
- deployment or coexistence

Summarize decisions after each group. Require explicit verification of high-impact decisions. Convert the result into acceptance criteria, non-goals, risks, and unresolved questions.
