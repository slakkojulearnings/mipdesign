# Purpose

Review repository structure and readiness.

# Prompt

Review canonical-file consistency, package boundaries, ignored confidential paths, setup commands, CI, tests, licensing, security files, generated artifacts, duplicate docs, stale instructions, and first-day developer experience. Provide severity, evidence, and surgical remediation.

# Required Output

- assumptions
- evidence used
- proposed or completed work
- tests and validation
- unknowns and risks
- next executable step

# Completion Rule

Do not declare completion until the stated acceptance criteria or validation commands pass. Follow `CLAUDE.md` and the relevant skill.
