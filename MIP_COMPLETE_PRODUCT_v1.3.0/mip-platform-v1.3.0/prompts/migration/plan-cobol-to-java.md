# Plan COBOL-to-Java Migration

Use the `cobol-to-java-migrator` skill.

Create a small, compartmentalized migration specification for the selected source program or workflow.

Include:

- source scope and excluded scope
- copybooks and record lengths
- files, datasets, tables, and external calls
- business rules and control flow
- target Java directory and package structure
- explicit codecs and I/O abstractions
- original-program harness
- unit, integration, byte-equivalence, edge-case, and performance tests
- risks, unknowns, and decision points
- verifiable completion criteria

Do not implement until the specification's key decisions are approved.
