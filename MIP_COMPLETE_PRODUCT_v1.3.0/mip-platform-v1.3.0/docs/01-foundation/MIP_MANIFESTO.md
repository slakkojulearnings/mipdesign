# MIP Manifesto

Legacy systems are not failures. They are durable repositories of business knowledge.

The modernization problem is not primarily a language problem. It is an understanding problem.

We therefore commit to:

- facts before AI
- evidence before conclusions
- understanding before transformation
- relationships before isolated artifacts
- business capability before technical decomposition
- deterministic verification before trust
- incremental modernization before big-bang replacement
- institutional memory that outlives individuals
- human review for consequential inferences
- preserving behavior while changing implementation

Source code is implementation. Knowledge is the asset. MIP exists to preserve that knowledge and make transformation safer.
