# MIP — Positioning Canvas (one page)

*Hand this to a stakeholder. Everything that matters, on a single sheet.*

---

**CATEGORY** &nbsp;|&nbsp; Modernization intelligence & verification backbone — a *system of record /
control plane* for legacy estates. **Not** a code generator.

**ONE-LINER** &nbsp;|&nbsp; *The only platform that tells you what your mainframe does — and proves
your replacement does the same thing.*

---

### THE PROBLEM
Modernization starts in the dark: estate undocumented, authors retired, dependencies hidden. Teams
**scope, sequence, and rewrite blind** → overruns, stalls, broken production. The bottleneck is
*understanding* the old code and *proving* the new code is equivalent — not writing it.

### THE WEDGE (why now)
AI made code generation **cheap** ⇒ **understanding + verification are the new bottleneck and the
value.** Ungoverned AI on a mainframe = *confident hallucination at scale.* **MIP is what makes AI
safe to use** — and its value *rises* as codegen commoditizes.

---

### WHAT IT DOES (lifecycle, not features)
`SEE` inventory + call/lineage/flow graphs → `UNDERSTAND` roots, customer journeys, capabilities →
`CAPTURE` business rules + evidence/confidence (preserve SME knowledge) → `DECIDE` service candidates
+ risk-ordered roadmap → `REBUILD` confirmed requirements → grounded Java → `PROVE` dual-run /
golden-master equivalence.

### THE MOAT (what AI tools can't claim)
1. **Evidence envelope** — every fact cited, confidence-scored, honest about uncertainty.
2. **Durable knowledge graph** — compounds across the program; outlives any model/vendor.
3. **Verifiable equivalence** — proves the Java behaves like the mainframe.
4. **The integration point** — LLMs/codegen/SIs are interchangeable workers; **MIP is the rails.**

---

### AUDIENCES → THE VALUE THEY BUY
| CIO / Board | De-risk; **capture retiring-SME knowledge**; spend on evidence |
| Enterprise Architect | Vendor-neutral control plane; safe decomposition |
| Delivery / SI | Discovery in weeks; safe strangler sequence |
| Risk / Compliance | Full traceability; proven behavior-equivalence |

### MIP vs. "another AI tool"
| AI tool | MIP |
|---|---|
| Generates plausible output | Records *proven*, cited facts |
| Hides what it doesn't know | Flags it (`needs_review` + confidence) |
| Disposable, one-shot | Persistent backbone the program runs on |
| "Trust me" | "Here's the source line — and the dual-run proof" |

---

### PROOF POINTS (demand these; MIP shows them)
Extraction **precision/recall** · estate **coverage %** · **% journeys SME-confirmed** ·
**equivalence pass-rate** (Java == mainframe) · every fact → file:line.

### THE DEMO THAT REFRAMES THE ROOM
Click an edge → cited source line + confidence. Then **dual-run**: mainframe output == Java output
**to the penny.** *Proof, not magic.* (Never demo "AI explains COBOL" — that's the toy framing.)

### THE ASK
Pilot **one customer journey** end-to-end — discover → rebuild → prove equivalent — against a
measurable equivalence-pass target. The whole thesis in one slice.
