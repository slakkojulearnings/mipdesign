01The wall

Nobody can read this at scale.

PROGRAM-ID. CRDPOST. 05 WS-BALANCE PIC S9(7)V99 COMP-3. *> money, "packed" 05 WS-STATUS PIC X. 88 ACCOUNT-OVERDUE VALUE 'O'. *> a business flag COPY CARDREC. *> a shared record layout PROCEDURE DIVISION USING LK-CARD-NO. *> receives a card number EXEC SQL SELECT BALANCE,STATUS INTO :WS-BALANCE,:WS-STATUS FROM CARD_MASTER WHERE CARD_NO=:LK-CARD-NO END-EXEC. IF ACCOUNT-OVERDUE COMPUTE WS-LATE-FEE = WS-BALANCE * 0.05 *> 5% late fee CALL 'FEEPOST' USING WS-LATE-FEE *> calls another program

Plain English: take a card number, look up its balance & status, and if overdue, charge a 5% late fee and hand it to FEEPOST. Now imagine 4,000 lines of this — and the author retired in 2003.

07The bridge to a new app

Facts → a confirmed rule → Java or Python.

Java — implements BR-007
if (!a.isOverdue()) return ZERO; return a.balance() .multiply(new BigDecimal("0.05")) .setScale(2, HALF_UP); // money, exact
Python — implements BR-007
if not account.overdue: return Decimal("0.00") return (account.balance * Decimal("0.05")) \ .quantize(Decimal("0.01"), ROUND_HALF_UP)

The packed COBOL number (COMP-3) becomes a BigDecimal / Decimal — not a float — so the math matches the mainframe to the penny. Each method cites the rule it implements.
