# Explain Repository Architecture

Analyze the repository using inventory, directory structure, source signatures, catalogs, and relationships.

Produce:

1. Repository purpose and supported business/technical capabilities
2. Directory tree with the purpose of each important folder
3. Artifact counts by type
4. Online entry paths
5. Batch entry paths
6. Data stores and interfaces
7. High-level architecture
8. Known daily/periodic workflows
9. Unknown or unresolved areas
10. Evidence and confidence

Do not infer organization-specific business meaning without source evidence.
