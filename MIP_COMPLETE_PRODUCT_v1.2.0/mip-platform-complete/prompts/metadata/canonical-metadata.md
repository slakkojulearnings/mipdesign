# Purpose

Create or update canonical metadata.

# Prompt

Define the smallest canonical change that represents the new source fact. Include stable identity, required attributes, evidence, confidence, validation status, lifecycle, backward compatibility, and examples. Update models, schema, exporters, graph ontology, API, tests, and canonical documentation together.

# Required Output

- assumptions
- evidence used
- proposed or completed work
- tests and validation
- unknowns and risks
- next executable step

# Completion Rule

Do not declare completion until the stated acceptance criteria or validation commands pass. Follow `CLAUDE.md` and the relevant skill.
