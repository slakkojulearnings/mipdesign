MIP v0.1
Objective

Given a folder containing:

mfcode/

├── cobol/
├── jcl/
├── copybooks/
└── db2/

Answer:

What programs exist?

What jobs exist?

What copybooks exist?

Which program calls which?

Which job executes which program?

Nothing else.

mip-platform/

        skill.md

Step 7 — Build Program Catalog

For COBOL only.

Extract:

PROGRAM-ID

Example:

PROGRAM-ID. PAY001.

Store:

{
  "program":"PAY001",
  "path":"..."
}
Copilot Prompt
Act as a Mainframe Modernization Expert.

Analyze COBOL source files.

Design a metadata extraction framework that extracts:

- PROGRAM-ID

Only.

Ignore CALLs.
Ignore COPYs.
Ignore SQL.

Return architecture only.

Notice the phrase Only.

That keeps scope under control.

Step 8 — Build JCL Catalog

Extract:

//PAYJOB JOB

and

EXEC PGM=

Store:

{
  "job":"PAYJOB",
  "program":"PAY001"
}
Step 9 — First Useful Query

You don't need Neo4j yet.

Use PostgreSQL.

Question:

SELECT *
FROM program_catalog
WHERE program_name='PAY001';

Then:

SELECT *
FROM job_program_relationship;

If you can answer:

Which jobs execute PAY001?

You've created the first real business value.

Your First Deliverable

After v0.1, you should be able to point MIP at a TSys repository and produce:

Programs Found: 12,842

Jobs Found: 2,315

Copybooks Found: 8,924

Relationships Found: 17,521

And answer:

Which jobs execute program X?

That may sound simple, but it's the first step from "source code" to "enterprise knowledge."

Once you reach that point, the next phase is CALL relationship discovery (Program → CALLS → Program), which is where the knowledge graph starts becoming powerful.
